Assumption #1

Tasks are added inside the task generator run function. I do this assuming that
tasks aren't added and removed often, so someone using these functions doesn't
need to add tasks themselves.

Assumption #2

We take in the output stream when we create the transmission thread. This allows
the user more flexibility in where their hex verification is stored and is
especially helpful for testing.

Assumption #3

If a tasks process function hasn't been run its processed value will be 0. This
is just to avoid undefined behaviour as under any normal circumstance the task 
will always be processed before transmission.
